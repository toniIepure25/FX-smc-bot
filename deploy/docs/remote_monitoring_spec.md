# Remote Monitoring Specification

## Design Principle

The operator should never need to SSH into the VPS during normal operation. All critical information flows to Telegram automatically.

## Monitoring Channels

### 1. Telegram Alerts (Real-time)
- **Trigger**: WARNING+ severity events from `AlertRouter`
- **Content**: Emoji-coded severity, category, message, timestamp
- **Latency**: < 10 seconds from event to notification
- **Silent**: INFO and WARNING alerts don't buzz; CRITICAL/EMERGENCY do

### 2. Daily Report (Scheduled, 22:00 UTC)
- Generated by `generate_report.py --type daily` via cron
- Sent to Telegram as a structured message
- Contains: trades, PnL, equity, drawdown, alerts, feed health, drift status

### 3. Weekly Report (Scheduled, Saturday 10:00 UTC)
- Generated by `generate_report.py --type weekly` via cron
- Comprehensive review: cumulative stats, daily breakdown, rejection pipeline, feed health

### 4. Health File (Continuous)
- `forward_runs/health.json` updated on state transitions
- Used by Docker HEALTHCHECK (every 60s)
- Can be polled via SSH or a simple HTTP health wrapper if needed

### 5. Structured Logs (On-demand)
- `forward_runs/logs/service.log` — full structured log
- `forward_runs/<run_id>/journal.jsonl` — event-level audit trail
- `forward_runs/alerts.jsonl` — all alerts (including INFO)
- Access via: `docker compose logs forward-paper` or SSH

## What Gets Monitored Automatically

| Signal | Source | Alert Level | Action |
|--------|--------|-------------|--------|
| Service start/stop | Service lifecycle | INFO | Telegram notification |
| Service crash | Exception handler | EMERGENCY | Telegram buzz + auto-restart |
| Feed gap > 51h | FeedHealthMonitor | WARNING | Telegram notification |
| Feed stale | FeedHealthMonitor | WARNING | Telegram notification |
| Risk state change | DrawdownTracker | WARNING/CRITICAL | Telegram notification |
| Circuit breaker fire | DrawdownTracker | CRITICAL | Telegram buzz |
| Drift detected | DriftDetector | WARNING | In daily report |
| Signal drought > 50 bars | Pipeline diagnostic | In report | Daily report flag |
| Config fingerprint mismatch | Startup validation | CRITICAL | Telegram buzz |
| Trade fill | PaperBroker | INFO (journal) | In daily report |
